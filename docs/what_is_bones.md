
Like Smalltalk, bones is about messaging and composition - the tree code (ast) comprises literals, type 
descriptions, name / value store and retrieve, function definitions, and message dispatches - just like 
Smalltalk there are no loops, or conditionals.

minimalist external DSL - pipeline syntax for wrangling and analysis
    immutable -> not good for transactions

kernel is embeddable (as a dll) and can be run within an agent process - via messaging

Like q/kdb bones is about messaging, high performance, partial application, composition and immutability. q/kdb terms 
frames as tables, bones terms tables as frames - they are the same thing.

Like q/kdb, and Smalltalk, bones is pithy and has a simple non-career programmer focused syntax.

Uniquely bones adds data-focussed types.
